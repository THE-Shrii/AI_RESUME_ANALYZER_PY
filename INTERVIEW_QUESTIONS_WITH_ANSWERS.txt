AI RESUME ANALYZER - INTERVIEW QUESTIONS WITH ANSWERS
=====================================================

Project Name: ResumeIQ - AI Resume Analyzer
Tech Stack: Python, Streamlit, Pandas, Plotly, PyPDF2, Gemini API, ReportLab, python-dotenv


1. What is the main objective of this project?
Answer:
The main objective of this project is to analyze a candidate's resume against a target job description and generate useful career insights. The application calculates ATS score, resume match score, skill gaps, matched skills, missing skills, AI resume feedback, interview questions, learning roadmap, recruiter summary, and a downloadable report.


2. Why did you build this project?
Answer:
I built this project to solve a real job-search problem. Many candidates do not know whether their resume matches a job description or how an ATS system may evaluate it. This project helps candidates improve their resume by identifying missing keywords, weak sections, and skill gaps. It also demonstrates my skills in GenAI, Python, Streamlit, data processing, prompt engineering, and product-style UI design.


3. What problem does this project solve?
Answer:
The project solves the problem of resume-job mismatch. Candidates often apply without knowing whether their resume contains the right skills, keywords, structure, or impact statements. ResumeIQ gives them an ATS score, match score, missing skills, improvement suggestions, and AI-generated preparation material so they can improve before applying.

5. Explain the architecture of your project.
Answer:
The project follows a modular architecture. The main `app.py` file controls the Streamlit layout, session state, sidebar uploads, and page navigation. The `pages/` folder contains separate UI modules for dashboard, interview questions, learning roadmap, and recruiter summary. The `utils/` folder contains reusable logic for PDF reading, text cleaning, skill extraction, ATS scoring, match scoring, Gemini analysis, and report generation. This keeps the application scalable and easier to maintain.


6. Why did you use Streamlit?
Answer:
I used Streamlit because it allows fast development of interactive data and AI applications using Python. It is suitable for dashboards, file upload workflows, charts, and AI demos. For this project, Streamlit helped me build a recruiter-friendly and portfolio-ready interface without needing a separate frontend framework.


7. How does resume text extraction work?
Answer:
The resume is uploaded as a PDF. The project uses PyPDF2 to read the PDF file page by page. For each page, it extracts available text and combines it into a single resume text string. This text is then passed to the cleaning, skill extraction, scoring, and AI analysis modules.


8. What happens if the PDF is scanned or image-based?
Answer:
PyPDF2 works best with text-based PDFs. If the PDF is scanned or image-based, text extraction may fail or return empty text. In the current version, the app shows a clear error message. A future improvement would be adding OCR support using tools like Tesseract, EasyOCR, or a cloud vision API.


9. How does text cleaning work?
Answer:
Text cleaning converts the text to lowercase, removes unnecessary special characters, normalizes whitespace, and prepares the text for keyword matching. This improves the accuracy of skill extraction because skills can be compared in a consistent format.


10. How does skill extraction work?
Answer:
The app loads a skill catalog from `data/skills.csv`. It scans the cleaned resume and job description text for known skills. If a skill from the catalog appears in the text, it is added to the detected skills list. The app then compares resume skills with job description skills to find matched and missing skills.


11. Why did you use a CSV file for skills?
Answer:
I used a CSV file because it is simple, editable, and scalable for a portfolio project. New skills can be added without changing the code. It also separates data from logic, which makes the project easier to maintain.


12. How is the resume match score calculated?
Answer:
The resume match score is calculated by comparing detected resume skills with detected job description skills. The formula is:

Match Score = (Number of Matched JD Skills / Total JD Skills) * 100

For example, if the job description has 10 detected skills and the resume contains 7 of them, the match score is 70%.


13. How is the ATS score calculated?
Answer:
The ATS score is calculated using multiple factors:
- Skill match
- Contact information
- Resume structure
- Impact signals
- Readability

The score is not based only on keywords. It also checks whether the resume has important sections like education, experience, projects, and skills, whether contact details are present, and whether the resume uses action-oriented impact words and measurable results.


14. What is the difference between ATS score and match score?
Answer:
The match score measures how well the resume skills match the job description skills. The ATS score is broader. It considers skill match, resume structure, contact information, readability, and impact signals. So match score is keyword-focused, while ATS score is resume-quality focused.


15. What are matched skills?
Answer:
Matched skills are skills that appear in both the resume and the job description. These indicate that the candidate already has some skills required for the role.


16. What are missing skills?
Answer:
Missing skills are skills found in the job description but not detected in the resume. These are important because they show what the candidate should learn or add to the resume if they already know them.


17. What are recommended skills?
Answer:
Recommended skills are generated from missing skills and important industry skills. They help the candidate understand which skills can improve their profile for the target role.


18. What are emerging industry skills?
Answer:
Emerging industry skills are modern, high-demand skills such as generative AI, LLM, RAG, vector databases, LangChain, prompt engineering, MLOps, and cloud deployment. These skills help make the candidate more competitive in current AI and software roles.


19. How did you integrate Gemini AI?
Answer:
I created a `GeminiAnalyzer` class inside `utils/gemini_analyzer.py`. It loads the API key from the `.env` file using `python-dotenv`, configures the Gemini model, and sends prompt templates for different tasks such as resume analysis, interview question generation, learning roadmap generation, and recruiter summary generation.


20. Why did you use environment variables?
Answer:
Environment variables keep sensitive information like API keys outside the source code. This is important for security and deployment. The Gemini API key is stored in `.env`, and `.env` is ignored by Git using `.gitignore`.


21. What happens if the Gemini API key is missing?
Answer:
The project has fallback responses. If the Gemini API key is missing or the API is unavailable, the app still generates useful deterministic content based on the analysis result. This makes the app demo-friendly and prevents the user experience from breaking.


22. What is prompt engineering in this project?
Answer:
Prompt engineering is used to instruct Gemini to generate structured outputs. For example, the resume analysis prompt asks for strengths, weaknesses, resume summary, improvement suggestions, and interview readiness score. Separate prompts are used for interview questions, learning roadmap, and recruiter summary.


23. Why did you create separate page files?
Answer:
Separate page files improve modularity. Instead of keeping all UI logic in one large file, the dashboard, interview questions, learning roadmap, and recruiter summary are separated into individual modules. This makes the app easier to understand, test, extend, and debug.


24. What is the role of `app.py`?
Answer:
`app.py` is the main entry point of the application. It configures the Streamlit page, loads CSS, initializes session state, renders the sidebar, handles file upload, builds the analysis result, and routes the user to different pages.


25. What is the role of session state?
Answer:
Session state stores analysis data and generated AI outputs across Streamlit reruns. Without session state, generated content like AI analysis or interview questions could disappear whenever the page refreshes or a user interacts with another widget.


26. Why did you use caching?
Answer:
Caching improves performance by avoiding repeated work. For example, extracted PDF text and the skill catalog can be reused instead of recalculated on every Streamlit rerun.
